# 演化世界 · 游戏化前期笔记

把 `evolve.html`(全球网格行星生命演化)做成一个游戏的前期决定、理念与待办。活文档,随讨论迭代。

## 一、项目定位(已定)
- 把 evolve 做成游戏。
- 引擎:**Godot**(开源、2D/3D皆可、能导出网页保留可达性;量级足够,将来要大规模可上 C#/GDExtension)。
- 画面:**像素**,程序化生成为主。
- 底层:**以现实科学体系为骨架**——物理/化学/生物/地质/生态的真实机制+守恒+涌现,自洽、互相联系。是科学的"道理"+合理简化,不是精确计算;我的科学知识有边界,关键处标注/数值验证兜底。尺度变,主导科学就变(细胞→生化,行星→地球系统科学,星系→天体物理)。

## 二、协作模型(核心理念)
- **用户 = 设定总指挥**:握源头(核心设定、世界是什么、往哪走),大方向的迭代由用户拍板。
- **我 = 底层引擎 + 扩写者**:主动生发底层物理逻辑/机制/细节,保持自洽、互相咬合成一个"设定整体";日常扩写、润色、执行归我。底层逻辑我自己推,不等用户告诉我"该怎样"。
- **脑子放开生发,手上落地谨慎**:想法/方案大胆推;但写文件、代码、提交前等用户点头(见"没同意不执行")。

## 三、工作方式偏好
- 回复:短、口语、纯文字,不用表格,不外露思考过程。
- 没同意不执行:有副作用的动作(写/改/删、跑命令、提交、新建)先经用户同意;纯只读探查不限。
- 纪律:先 headless/数值验证再落地(沿用 evolve 的做法,搬到 Godot:GDScript 写逻辑 → Godot headless 验证 → 编辑器看效果)。

## 四、核心体验与目标(已定 2026-06-23)
**这是个生存游戏。** 之前搭的全球演化星球只是它的"活世界"底层。

- **玩家是谁**:一个人(人体生理模型当内核,见 `design_food_water.md`:水/糖原/脂肪/胃/电解质,症状而非血条,机能输出而非掉 HP)。
- **在哪求生**:这颗活星球的某个地点(局部地点图,全球 Planet 每天喂气候/海温/季节)。参照 **Card Survival** 的生理深度,但**世界是确定性、涌现、还会自己演化**的。
- **核心幻想(B 打底 + D 精神)**:赤手空拳,靠**看懂世界怎么运作**,从"勉强不死"爬到"繁荣定居"。进展不是抽卡解锁,是"理解机制 + 存量攒够跨阈 → 能力涌现"。

**D 的精神(守零随机)**:世界可复现、死亡有因可循(兔肉饥饿/断水/失温)→ 玩家从后果学习不怪运气。"知识=进展"两层:① 玩家脑里的知识(认植物/读天象)靠观察积累;② 角色能力(配方/技艺)由"理解+存量跨阈"确定性解锁。灾难有**确定性前兆**(冰期前降温趋势、洋流变化)→ 看懂的人提前备粮/迁徙,这是通往 A/C 的桥。

**进展阶梯(B,由内到外)**:① 活命(水→食→体温,症状驱动)② 火与工具(确定性取火/石器/容器/烹饪)③ 庇护与微气候(挡外部热负荷,熬夜寒/换季)④ 储备与规划(读季节/前兆→晒干储粮储水,扛坏季节)⑤ 驯化与定居(跟演化出的物种互动,觅食变生产→繁荣)。

**后期目标(已排在 B 之后)**:
- **A 世系存续**:定居后繁衍,时间拉到地质级,冰期/群系迁移/大灭绝开始咬人,家族跨世代适应。生存→演化的接续。
- **C 旅程/逃脱**:明确大目标——跨越星球、追随群系迁移、逃离恶化地带。把 A 的深时间压力变成有终点的远征。

**底层接口**:全球演化星球(Godot 已搭)喂气候/季节/灾难;演化出的物种成为可觅食/驯化的动植物;人体模型当生理内核。

**下一步**:做"能证明魔法的最小可玩切片"——一个人在一个地点,身体模型(水/食/温)接上活气候,靠理解熬过一个季节。

**仍没定**:世界设定根基(严格地球 vs 科学框架内架空)。

## 五、前期事项框架(由内核到外延,越内核越先定)
- **第0层 愿景与意图**:核心情感(敬畏/造物/对涌现的惊叹/深时间)、主题隐喻、一句话愿景。
- **第1层 玩家与吸引力**:目标玩家、差异化钩子(对比 Spore/Thrive/SimEarth)、可讲述性、重玩与长尾。
- **第2层 中心设计问题**:演化是慢的、自动的,怎么变成"玩家有事做、有乐趣"+ 玩家能动性(上帝/物种/选择压)+ 核心循环 + 进展结构 + 真实 vs 好玩的线。
- **第3层 体验与可读性**:美学方向、信息设计(谱系树/图层/时间线)、时间的"摄像机"(把百万年压成有节奏)、onboarding。
- **第4层 制作与可持续**:北极星/最小垂直切片、技术路线、内容生产管线(程序化生成是天然盟友)、协作模型、预算/发布、风险登记。
- **贯穿**:先找乐趣(最丑原型验核心魔法)、砍范围的勇气、为涌现留地、警惕"模拟者陷阱"(沉迷系统深度忘了乐趣)。
- **先抓三件**:核心情感与幻想 / 中心设计问题 / 能证明魔法的最小可玩切片。

## 七、性能(已评估)
- 模拟一步(JS):3528格=0.13ms,14000格=0.54ms,5-20万格才几ms。成本 O(N) 线性,无指数爆炸。
- 真正成本是"格子数 × 每秒模拟步数";演化慢,模拟可低频跑、与渲染解耦 → 成本几乎可忽略。
- Godot:渲染用纹理/TileMap/shader(别用几千节点);GDScript 慢 JS 约 10-30倍,大网格高频跑就改 C#/GDExtension;模拟分辨率与显示分辨率可解耦(模拟粗、显示细)。
- 结论:现在格子数随便加,按"看着舒服"定即可。
